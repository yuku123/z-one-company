# Z-MQ 技术架构文档

## 1. 项目概述

Z-MQ 是一个高性能、高可靠的分布式消息队列系统，对标 RocketMQ，支持亿级消息堆积、高吞吐量、低延迟的消息传递。

### 1.1 核心特性

- **高吞吐量**: 单机可达10万+ TPS
- **低延迟**: 消息投递延迟 < 10ms
- **高可靠**: 支持同步/异步刷盘，消息不丢失
- **高可用**: 主从架构，自动故障转移
- **海量堆积**: 支持亿级消息堆积
- **灵活消费**: 支持集群消费、广播消费、顺序消费

## 8. 关键技术决策

### 8.1 存储选型对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **Mmap+PageCache** (选中) | 性能高、实现简单 | 依赖系统调页 | 通用场景 |
| **DIO (Direct I/O)** | 绕过PageCache | 实现复杂、兼容性差 | 特殊场景 |
| **RocksDB** | 功能丰富、可靠 | 依赖第三方、调优复杂 | 需要KV存储 |

### 8.2 复制策略对比

| 策略 | 数据可靠性 | 性能 | 延迟 | 适用场景 |
|------|------------|------|------|----------|
| **同步双写** | 最高 | 低 | 高 | 金融级场景 |
| **异步复制** (默认) | 高 | 高 | 低 | 通用场景 |
| **DLedger Raft** | 高 | 中 | 中 | 自动故障转移 |

### 8.3 技术栈选型

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 编程语言 | Java 17+ | 现代化语法，性能优化 |
| 构建工具 | Maven | 多模块管理 |
| 网络通信 | Netty 4.1.x | 高性能NIO框架 |
| 序列化 | JSON / Protobuf | 灵活可扩展 |
| 存储 | Mmap + FileChannel | 零拷贝高性能 |
| 日志 | SLF4J + Logback | 标准日志方案 |
| 测试 | JUnit 5 + Mockito | 单元测试 |

## 9. 性能指标

### 9.1 目标性能

| 指标 | 目标值 | 测试条件 |
|------|--------|----------|
| 写入TPS | 10万+/s | 单机，异步刷盘 |
| 读取TPS | 10万+/s | 单机，从ConsumeQueue读取 |
| 端到端延迟 | < 10ms | P99，同步刷盘 |
| 消息堆积 | 10亿+ | 磁盘充足 |
| 系统可用性 | 99.99% | 集群部署 |

### 9.2 资源消耗

| 资源类型 | 建议配置 | 说明 |
|----------|----------|------|
| CPU | 8核+ | 消息处理、网络IO |

| 磁盘 | SSD / NVMe | 顺序写入性能 |
| 网络 | 万兆网卡 | 高吞吐场景 |
