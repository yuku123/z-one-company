# 001_项目总览

## 1.1 模块定位

z-mq（消息中心）是系统的异步消息中枢，为各模块提供可靠的消息发布/订阅、事件通知、系统解耦能力。

## 1.2 愿景

- **主题管理**：消息主题/队列的创建、配置、监控
- **消息发布订阅**：发布消息到指定主题，消费者按需订阅
- **消息持久化**：消息可靠存储，支持消费重试
- **死信队列**：消费失败的消息进入 DLQ，便于排查
- **延迟消息**：支持指定延迟时间再投递（订单超时取消等场景）

## 1.3 模块关系

```
z-wf（审批完成通知）───► z-mq ──► z-task（创建跟进任务）
z-task（任务完成）───► z-mq ──► z-wf（更新审批状态）
z-schedule（任务执行失败）───► z-mq ──► z-ctc（发送告警通知）
z-ctc-audit-starter（审计事件）───► z-mq ──► z-ctc（异步存储）
```

## 1.5 消息模型

```
Exchange（交换机）
  └── Queue（队列）───► Consumer（消费者）
         └── Binding（绑定规则：routing key 匹配）
```

|--------|------|
| P0 | RabbitMQ 部署、Topic/Queue 管理、消息发送与消费 |
