# 001_项目总览

## 1.1 模块定位

z-gw（Gateway）是**高性能 API 网关**，作为微服务架构的统一入口，提供请求路由、负载均衡、限流熔断、认证鉴权等核心能力。基于 Netty 构建，实现全异步非阻塞架构。

## 1.2 愿景

为一人公司提供企业级的 API 网关能力：
- **高性能**：基于 Netty 的异步非阻塞架构，单机支持 10万+ QPS
- **动态路由**：支持基于 URL、Header、Query 参数的路由匹配
- **限流熔断**：令牌桶、滑动窗口等限流算法，自动熔断降级
- **插件扩展**：基于责任链的过滤器模式，支持自定义插件
- **配置热更新**：运行时配置变更，无需重启

## 1.3 模块关系

```
客户端 ──► z-gw（网关）──鉴权&路由──► 后端微服务
                        │
                  z-ctc（Token 验证）
```

## 1.5 模块结构

```
z-gw/
├── z-gw-core/              # 核心模块
│   ├── common/             # 公共工具类
│   ├── server/             # Netty HTTP 服务器
│   ├── router/             # 路由模块
│   ├── loadbalance/        # 负载均衡
│   └── filter/             # 过滤器链
├── z-gw-auth/               # 认证鉴权模块
├── z-gw-ratelimit/         # 限流熔断模块
├── z-gw-discovery/         # 服务发现模块
├── z-gw-monitor/           # 监控告警模块
├── z-gw-admin/             # 管理控制台
└── z-gw-starter/           # 启动入口
```

## 1.6 核心功能

| 功能 | 说明 |
|------|------|
| HTTP Server | 基于 Netty 的高性能 HTTP 服务器 |
| 路由引擎 | 动态路由、路由热更新、重试机制 |
| 负载均衡 | 轮询、随机、加权、最少连接、IP Hash |
| 限流熔断 | 令牌桶、漏桶、滑动窗口、熔断降级 |
| 认证鉴权 | JWT、OAuth2.0、API Key、Basic Auth |
| 插件机制 | SPI 扩展、内置日志/监控/缓存/CORS |
