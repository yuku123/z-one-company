# 007_开发计划

## P0 - 核心能力（必须完成后再上线）

| 功能 | 说明 | 状态 |
|------|------|------|
| HTTP 路由 | 接收请求，根据路径前缀转发到对应后端服务 | ✅ 基础完成 |
| 服务注册/发现 | 各模块启动时注册到 z-gw，z-gw 动态感知服务列表 | ⚠️ 模式B（内置），模式A（Consul）规划中 |
| 健康检测 | z-gw 定期 ping 后端服务，自动摘除不健康节点 | ❌ 待开发 |
| 负载均衡 | Random/RoundRobin/加权路由策略 | ❌ 待开发 |
| JWT 鉴权 | 网关统一验证 Token，Token 无效/过期返回 401 | ⚠️ AuthHandler 存在，需完善 |
| 统一响应 | 后端返回结构统一化（code/message/data） | ❌ 待开发 |
| 请求日志 | 记录每个请求的 path/cost/status | ⚠️ GlobalFilter 存在，需完善 |

## P1 - 生产就绪

| 功能 | 说明 |
|------|------|
| 限流 | 令牌桶限流，支持按服务/用户/全局维度 |
| 熔断降级 | 后端服务持续超时/错误，自动熔断，返回降级响应 |
| Consul / Nacos 兼容 | 对接外部成熟注册中心，替代内置注册 |
| z-gw 集群 | 多网关节点部署，避免单点 |
| 动态配置 | 路由规则/限流阈值热更新，无需重启 |

## P2 - 高阶特性

| 功能 | 说明 |
|------|------|
| gRPC 路由 | 支持 gRPC 协议转发 |
| WebSocket 支持 | 支持 WebSocket 长连接代理 |
| 流量染色 | 请求头打标，实现测试流量和正式流量分离 |
| 插件化过滤器 | 通过 z-ext 动态加载自定义过滤器 |
| 请求/响应改写 | URL 重写、请求头/响应头增删改 |
