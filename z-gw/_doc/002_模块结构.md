# 002_模块结构

## 2.1 代码目录

```
z-gw/
└── z-gw-starter/                       # Netty 网关核心
    └── src/main/java/com/zifang/zwg/starter/
        ├── Server.java                  ← Netty ServerBootstrap 启动
        ├── handler/
        │   ├── HttpServerCodec.java     ← HTTP 编解码
        │   ├── AggregatorHandler.java   ← 请求体聚合（POST Body）
        │   ├── AuthHandler.java         ← JWT 鉴权
        │   ├── RouterHandler.java       ← 路由转发到后端服务
        │   └── ProxyBizHandler.java    ← 代理业务处理器
        ├── router/
        │   └── ServiceRouter.java      ← 动态路由（根据路径分发到不同服务）
        └── filter/
            ├── GlobalFilter.java       ← 全局过滤器（链路追踪、日志）
            ├── RateLimitFilter.java    ← 限流过滤器
            └── CircuitBreakerFilter.java # 熔断降级过滤器
```

## 2.2 注册中心架构（两种模式）

**模式 A（推荐）：外部注册中心**

```
Consul / Nacos（外部）
  └── 注册：z-task:8090, z-wf:8091, z-ctc:8092, ...
        └── z-gw 从注册中心拉取服务列表，动态感知上下线
```

**模式 B：内置注册中心**

```
z-gw 内置轻量注册中心
  └── 各模块启动时向 z-gw 注册
        └── z-gw 维护内存服务列表（简单，轻量，适合小规模）
```

当前代码处于 **模式 B** 阶段，模式 A（Consul 兼容）规划中。

## 2.3 核心模块说明

| 模块 | 说明 |
|------|------|
| **Server** | Netty NIO 服务端，监听端口，维护 Channel |
| **HttpServerCodec** | HTTP 请求/响应编解码 |
| **AggregatorHandler** | 将分段 Chunked 请求聚合为完整 ByteBuf |
| **AuthHandler** | JWT Token 验证，Token 失效/格式错误返回 401 |
| **RouterHandler** | 根据请求路径从服务列表中选择目标主机，发起代理请求 |
| **ProxyBizHandler** | 接收代理响应，写回客户端 |
| **RateLimitFilter** | 基于令牌桶/滑动窗口的限流（单机） |
| **CircuitBreakerFilter** | 熔断降级（某服务持续失败时暂时跳过） |

## 2.4 请求处理流程

```
Client 请求
  └── Netty Boss 接收
        └── Pipeline:
            ├── HTTP Codec（解码）
            ├── Aggregator（聚合 Body）
            ├── Auth（JWT 验证）
            ├── RateLimit（限流检查）
            ├── CircuitBreaker（熔断检查）
            └── Router（转发到后端服务）
                  └── Proxy（接收响应，写回 Client）
```

## 2.5 路由规则

当前路由基于**路径前缀 + 服务名**匹配：

```
/api/task/*   → z-task:8090
/api/wf/*     → z-wf:8091
/api/ctc/*    → z-ctc:8092（CTC 独立端口）
/api/*        → z-config:8848（默认配置中心）
```
