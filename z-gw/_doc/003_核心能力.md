# 003_核心能力

## 3.1 高性能 HTTP Server

基于 Netty 构建的异步非阻塞 HTTP 服务器，提供高性能请求接入能力。

| 特性 | 说明 |
|------|------|
| 异步非阻塞 | 基于 Netty EventLoop 的全异步架构 |
| 连接管理 | Keep-Alive、零拷贝、内存池复用 |
| 线程模型 | Boss-Reactor + Worker-Reactor |

**配置参数**：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| server.port | 监听端口 | 8080 |
| server.bossThreads | Boss线程数 | 1 |
| server.workerThreads | Worker线程数 | 0 (CPU cores × 2) |
| server.soBacklog | TCP backlog | 1024 |
| server.maxContentLength | 最大内容长度 | 10MB |

## 3.2 动态路由

支持基于 URL 路径、HTTP 方法、Header 参数的灵活路由匹配。

| 特性 | 说明 |
|------|------|
| 路径匹配 | Ant 风格路径模式 (`/api/users/**`) |
| 方法匹配 | 支持 GET/POST/PUT/DELETE/* |
| Header路由 | 基于请求头的条件路由 |
| 动态更新 | 路由规则运行时热生效 |

**路由配置示例**：

```yaml
router:
  routes:
    - id: user-service
      path: /api/users/**
      method: "*"
      backend: http://localhost:8081
      stripPrefix: true
```

## 3.3 负载均衡

内置多种负载均衡算法，支持后端服务实例的智能选择。

| 算法 | 说明 |
|------|------|
| RoundRobin | 轮询（默认） |
| Random | 随机 |
| Weighted | 加权 |
| LeastConnections | 最少连接 |
| IpHash | IP哈希 |

## 3.4 限流熔断

多维度限流保护和熔断降级能力，保护系统稳定性。

### 限流

| 特性 | 说明 |
|------|------|
| 算法支持 | 令牌桶、滑动窗口、漏桶 |
| 限流维度 | IP、用户、API、全局 |
| 响应头 | X-RateLimit-Limit、Retry-After |

### 熔断

| 特性 | 说明 |
|------|------|
| 熔断策略 | 基于错误率、响应时间 |
| 半开状态 | 探测恢复 |
| 降级处理 | 兜底响应 |

## 3.5 认证鉴权

多协议认证支持，可扩展的认证Provider机制。

| 认证方式 | 说明 |
|----------|------|
| Bearer (JWT) | OAuth2.0 Bearer Token |
| Basic | HTTP Basic 认证 |
| API Key | API Key 认证 |

**公开路径配置**：

```yaml
auth:
  publicPaths:
    - /health
    - /healthz
    - /actuator/health
    - /api/v1/auth/login
```

## 3.6 过滤器链

基于责任链模式的插件化过滤器架构，支持自定义扩展。

**内置过滤器**：

| 过滤器 | 优先级 | 说明 |
|--------|--------|------|
| AuthFilter | 20 | 认证鉴权 |
| RateLimitFilter | 50 | 限流控制 |
| LoggingFilter | 100 | 请求日志 |

**扩展机制**：通过实现 `Filter` 接口并注册到 `FilterManager` 即可自定义过滤器。

## 3.7 配置热更新

支持运行时配置变更，无需重启网关即可生效。

| 配置类型 | 更新方式 |
|----------|----------|
| 路由规则 | 动态推送 |
| 限流参数 | 动态推送 |
| 认证配置 | 动态推送 |
