# 001_项目总览

## 1.1 模块定位

z-mist（密钥管理中心）是所有模块的密钥安全存储与按需分发中枢。密钥（密码、API Key、证书、Token）在 mist 加密存储，各模块通过授权接口按需获取，彻底杜绝密钥硬编码。

## 1.2 愿景

- **全密钥生命周期**：创建 → 存储（加密）→ 分发 → 轮换 → 销毁
- **环境隔离**：同一密钥支持 dev/staging/prod 多套环境，按环境独立分发
- **操作审计**：每次密钥访问均记录审计日志（对接 z-ctc-audit-starter）
- **自动轮换**：支持计划性密钥轮换，无需重新部署

## 1.3 模块关系

```
z-ctc（认证）───► z-mist（获取密钥完成登录/签发 Token）
z-task/z-wf/z-config 等模块 ──► z-mist（获取数据库密码、第三方 API Key）
z-mist（密钥变更通知）───► z-ctc-audit-starter（审计日志）
```

## 1.6 现有代码

**Controller**：`SecretController`（`secret/add`, `secret/{id}`, `secret/{id}/delete`）

**实体**：`Secret`（密钥基本信息）、`SecretVersion`（历史版本）、`SecretAccessLog`（访问日志）

**状态**：骨架存在，功能待完善。密钥加密存储、自动轮换、访问控制尚未实现。

## 1.7 开发优先级

| 优先级 | 内容 |
|--------|------|
| P0 | SecretController CRUD、密钥 AES 加密存储、环境隔离 |
| P1 | 密钥访问鉴权（哪些模块可以取哪个密钥）、访问日志 |
| P2 | 自动轮换计划、Web Console 管理界面 |
