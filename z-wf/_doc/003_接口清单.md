| 编号 | 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|------|

| 参数名 | 类型 | 必填 | 说明 |

### 2.8 启动流程实例

| 编号 | 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|------|
| 008 | 启动流程实例 | POST | `/approval-center/processes/start` | 启动一个新的流程实例 |

**请求体：**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| processKey | String | 是 | 流程定义Key |
| businessKey | String | 否 | 业务关联Key |
| initiator | String | 否 | 发起人 |
| title | String | 否 | 流程标题 |
| variables | Map | 否 | 流程变量 |

**请求示例：**
```json
{
  "processKey": "leaveProcess",
  "businessKey": "LEAVE-2024-001",
  "initiator": "zhangsan",
  "title": "张三请假申请",
  "variables": {
    "days": 3,
    "reason": "生病请假"
  }
}
```

**响应示例：**
```json
{
  "code": 200,
  "data": {
    "processInstanceId": "xxx",
    "businessKey": "LEAVE-2024-001",
    "message": "流程启动成功"
  }
}
```

## 5. 接口响应格式

### 5.1 统一响应结构

所有接口统一使用 `Result<T>` 包装响应：

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | 状态码：200=成功，其他=失败 |
| msg | String | 消息 |
| data | T | 数据内容 |

### 5.2 分页响应结构

分页接口使用 `PageResult<T>` 包装：

| 字段 | 类型 | 说明 |
|------|------|------|
| list | List<T> | 数据列表 |
| total | Long | 总记录数 |
| pageNum | Integer | 当前页码 |
| pageSize | Integer | 每页大小 |
