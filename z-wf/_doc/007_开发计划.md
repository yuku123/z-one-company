# 007_开发计划

## P0 - 审批核心（上线前必须完成）

| 功能 | 说明 | 状态 |
|------|------|------|
| 流程定义解析 | JSON/YAML 定义文件解析，支持会签/或签/条件路由 | ❌ 待开发 |
| 流程实例管理 | 启动、暂停、恢复、终止、删除 | ⚠️ 基础骨架有，需完善 |
| 节点路由 | 条件分支（金额/部门/职级/申请人） | ❌ 待开发 |
| 审批操作 | 同意、拒绝、转交、撤回、加签、减签 | ❌ 待开发 |
| 审批历史 | 完整审批链路记录（审批人/时间/意见/结果） | ❌ 待开发 |
| 审批中心统一接口 | `/approval-center/*` 通用审批 API | ⚠️ 接口已有，需和前端对齐 |

## P1 - 生产就绪

| 功能 | 说明 |
|------|------|
| 流程超时处理 | 节点超时未审批，自动催办或自动通过/拒绝 |
| 流程版本管理 | 流程定义修改后新建版本，运行中实例继续用旧版 |
| 回调通知 | 审批完成后通过 z-mq 回调发起方（z-task 等） |
| Camunda 7 兼容 | 如果自研困难，可考虑接入 Camunda 引擎 |

## P2 - 高阶特性

| 功能 | 说明 |
|------|------|
| 审批表单可视化设计器 | 前端拖拽配置审批字段 |
| 流程模拟运行 | 上线前模拟流程走向 |
| 流程监控大盘 | 各部门待办/已办/超时的统计看板 |
| 与 z-task 深度集成 | 审批通过自动在 z-task 创建任务，指派给审批人 |
