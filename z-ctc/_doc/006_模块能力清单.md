# 模块能力清单

> 整理各模块的功能优先级和实现状态，供后续开发参照。

## 1. 平台基座层

### z-ctc（4A 认证中心）

| 能力 | 说明 | 优先级 |
|------|------|--------|
| 用户名密码登录 | 用户名+密码认证，返回 JWT | P0 ✅ |
| 用户注册 | 用户名/手机/邮箱注册 | P0 ✅ |
| 发送验证码 | 通用验证码发送 | P0 ✅ |
| 重置密码 | 手机/邮箱验证码重置密码 | P0 ✅ |
| JWT Token 签发与验证 | Token 签发、刷新、主动失效 | P0 ✅ |
| 角色管理 | 角色 CRUD，角色关联权限 | P0 ✅ |
| 权限管理 | 权限 CRUD，菜单/按钮级别 | P0 ⚠️ 菜单树/按钮级未完善 |
| 用户角色关联 | 用户-角色多对多 | P0 ✅ |
| 操作审计日志 | 记录所有敏感操作 | P0 ✅ |
| 多租户隔离 | 对内一个租户，对外每产品线一个租户 | P1 ⚠️ |
| 域管理 | 租户下划域（运营域、用户域等） | P1 ⚠️ |
| 短信/邮箱验证码登录 | 手机号+验证码、邮箱+验证码 | P1 ✅ |
| 单点登录 SSO | 跨系统 Session 共享 | P2 ❌ |
| 外部 OAuth2 | 对接 GitHub/微信等第三方登录 | P2 ❌ |

### z-config（配置中心）

| 能力 | 说明 | 优先级 |
|------|------|--------|
| 配置存储 | MySQL 存储配置项，环境+应用+key 维度 | P0 ✅ |
| 配置读取 | API 获取当前配置 | P0 ✅ |
| 多环境隔离 | dev / test / prod 环境配置隔离 | P0 ✅ |
| 配置版本管理 | 配置变更历史，支持回滚 | P1 ✅ |
| 配置变更推送 | 配置变更主动推送到订阅服务 | P1 ⚠️ |
| 本地配置兜底 | 无 z-config 时读取本地 application.yml | P0 ✅ |

### z-gw（API 网关）

| 能力 | 说明 | 优先级 |
|------|------|--------|
| HTTP 路由 | 按路径前缀路由到后端服务 | P0 ✅ |
| 服务注册/发现 | 服务启动时注册，z-gw 维护注册表 | P0 ⚠️ in-memory |
| 健康检测 | 心跳检测，自动摘除不健康实例 | P0 ⚠️ |
| 负载均衡 | 随机/轮询/加权/最少连接 | P0 ⚠️ |
| 统一鉴权 | JWT Token 校验 | P0 ⚠️ |
| 统一响应格式 | 全局响应包装（code/message/data） | P0 ⚠️ |
| 请求日志 | 完整请求链路日志 | P0 ✅ |
| 限流 | 基于服务/IP/用户维度的限流 | P1 ❌ |
| 熔断降级 | 下游异常时快速失败+兜底返回 | P1 ❌ |
| Consul 注册中心兼容 | 模式A：注册到外部 Consul | P1 ❌ |
| z-gw 集群 | 多实例部署，注册数据共享 MySQL | P1 ❌ |

## 2. 运维/开发平台层

### z-task（任务中心）

| 能力 | 说明 | 优先级 |
|------|------|--------|
| 任务 CRUD | 创建/查询/修改/删除任务 | P0 ✅ |
| 任务分配 | 指定负责人 | P0 ✅ |
| 任务状态机 | 创建→进行中→待验收→已完成/已取消 | P0 ✅ |
| 任务优先级 | 紧急/高/中/低 | P0 ✅ |
| 触发方式 | 手动/API/定时（z-schedule）触发 | P0 ✅ |
| 关联审批流 | 任务可绑定 z-wf 审批流 | P1 ❌ |
| 任务评论 | 负责人/协作人员评论 | P1 ❌ |
| 任务附件 | 文件上传/下载 | P1 ❌ |
| 任务依赖 | 阻塞关系（A 完成后 B 才能开始） | P1 ❌ |
| 截止时间提醒 | 倒计时，超时提醒 | P1 ❌ |

### z-wf（工作流/审批中心）

| 能力 | 说明 | 优先级 |
|------|------|--------|
| 流程定义 | JSON/YAML 定义流程结构（节点、连线、条件） | P0 ❌ |
| 流程实例 | 启动流程，生成实例，记录历史 | P0 ❌ |
| 审批节点 | 会签、或签、条件路由 | P0 ❌ |
| 审批操作 | 同意/拒绝/转交/撤回/加签 | P0 ❌ |
| 审批历史 | 完整审批轨迹记录 | P0 ❌ |
| 流程抄送 | 审批结果抄送给相关人 | P1 ❌ |
| 流程超时 | 节点超时提醒/自动通过/自动拒绝 | P1 ❌ |
| 流程版本 | 流程定义变更后，老实例继续按旧版本执行 | P1 ❌ |
| 回调通知 | 流程节点完成后回调外部系统 | P1 ❌ |

### z-schedule（调度中心）

| 能力 | 说明 | 优先级 |
|------|------|--------|
| Cron 表达式 | 标准 Cron 定时触发 | P0 ✅ |
| 分布式任务 | 多实例部署时，保证任务不重复执行 | P0 ⚠️ |
| 失败重试 | 任务失败后按策略重试 | P0 ✅ |
| 任务执行日志 | 记录任务执行时间、耗时、结果 | P0 ✅ |
| 调度任务管理 | 任务的增删改查，启用/禁用 | P0 ✅ |
| 任务分发 | 一个任务分发给多个执行节点 | P1 ❌ |
| 任务依赖 | 任务 A 执行完成后，任务 B 才能执行 | P1 ❌ |
| 暂停/恢复 | 手动暂停调度任务 | P1 ❌ |
| 手动触发 | 手动立即执行一次定时任务 | P1 ✅ |

## 3. 中间件平台层

### z-mq（消息队列）

| 能力 | 说明 | 优先级 |
|------|------|--------|
| 消息持久化 | 消息存储到磁盘，防止服务重启丢失 | P0 ✅ |
| P2P 模型 | 点对点队列，一个消息只能被一个消费者消费 | P0 ✅ |
| Pub/Sub 模型 | 发布订阅，一个消息被所有订阅者消费 | P0 ✅ |
| 消息确认 | 消费者处理成功后确认，防止消息丢失 | P0 ✅ |
| 本地内存兜底 | 外部 MQ 不可用时，使用内存 EventBus | P0 ✅ |
| 死信队列 | 处理失败的消息进入 DLQ | P1 ❌ |
| 消息重试 | 失败消息按策略重试（指数退避） | P1 ✅ |
| Kafka 兼容 | 后期兼容 Kafka 协议 | P2 ❌ |

### z-rpc（RPC 框架）

| 能力 | 说明 | 优先级 |
|------|------|--------|
| 同步调用 | 请求-响应模式 | P0 ✅ |
| 负载均衡 | 随机/轮询/加权/一致性哈希 | P0 ✅ |
| 失败重试 | 调用失败按策略重试 | P0 ✅ |
| 序列化 | JSON / Protobuf 可选 | P0 ✅ |
| HTTP 降级 | z-rpc 不可用时，降级为 HTTP via z-gw | P0 ✅ |
| 异步调用 | Future / Callback 模式 | P1 ❌ |
| 单向调用 | Fire & Forget，不等待响应 | P1 ❌ |
| 熔断器 | 下游异常时快速失败 | P1 ❌ |

### z-ext（业务扩展）

| 能力 | 说明 | 优先级 |
|------|------|--------|
| 插件化扩展点 | SPI 机制，业务可注入自定义实现 | P0 ✅ |
| 业务策略注入 | 业务流程中可插拔的业务策略 | P0 ✅ |
| 统一拦截层 | 统一响应包装、统一异常处理、日志切面 | P0 ✅ |
| 逻辑热插拔 | 业务逻辑动态加载/卸载 | P1 ❌ |
| 链路追踪 | TraceId 透传，请求链路可追踪 | P1 ❌ |
