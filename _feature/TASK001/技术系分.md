# TASK001 技术实现方案

## 一、需求概述

实现用户注册、登录、密码找回功能，支持以下场景：
- **注册**：手机注册、邮箱注册、用户名/密码注册
- **登录**：手机登录、用户名登录
- **找回**：手机密码找回、邮箱密码找回

> 微信注册暂搁置

## 二、架构说明

- **z-msg**：负责对外发送通知（短信、邮件）
- **z-meta**：管理短信/邮件等通知资源（如短信通道、邮箱配置）
- **z-ctc**：用户中心，负责用户实体、认证、授权逻辑
- **z-gw**：网关层，负责路由和认证拦截

### 3.2 验证码表 (sys_verify_code)

新建表，用于存储验证码记录：

```sql
CREATE TABLE `sys_verify_code` (
  `id` bigint NOT NULL AUTO_INCREMENT COMMENT '主键',
  `biz_type` varchar(32) NOT NULL COMMENT '业务类型: REGISTER-注册, LOGIN-登录, RESET_PWD-重置密码',
  `code_type` varchar(32) NOT NULL COMMENT '验证码类型: PHONE-手机, EMAIL-邮箱',
  `receiver` varchar(128) NOT NULL COMMENT '接收者(手机号/邮箱)',
  `code` varchar(16) NOT NULL COMMENT '验证码',
  `expire_time` datetime NOT NULL COMMENT '过期时间',
  `used` tinyint NOT NULL DEFAULT '0' COMMENT '是否已使用: 0-否, 1-是',
  `gmt_create` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `gmt_modified` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  KEY `idx_receiver_expire` (`receiver`, `expire_time`)
) ENGINE=InnoDB COMMENT='验证码记录表';
```

**对应实体类**：
- 在 `z-ctc/z-ctc-core/src/main/java/com/zifang/ctc/core/domain/entity/VerifyCode.java`

## 四、接口设计

### 4.1 注册相关

#### 4.1.1 发送注册验证码
- **接口**: `POST /api/ctc/user/register/send-code`
- **请求体**:
```json
{
  "receiver": "13800138000",  // 手机号或邮箱
  "type": "PHONE"  // PHONE | EMAIL
}
```
- **响应**:
```json
{
  "code": 0,
  "message": "发送成功",
  "data": null
}
```

#### 4.1.2 手机注册
- **接口**: `POST /api/ctc/user/register/phone`
- **请求体**:
```json
{
  "phone": "13800138000",
  "code": "123456",
  "password": "xxx"
}
```

#### 4.1.3 邮箱注册
- **接口**: `POST /api/ctc/user/register/email`
- **请求体**:
```json
{
  "email": "test@example.com",
  "code": "123456",
  "password": "xxx"
}
```

#### 4.1.4 用户名密码注册
- **接口**: `POST /api/ctc/user/register/username`
- **请求体**:
```json

  "username": "testuser",

}
```

## 六、流程设计

### 6.1 手机注册流程
```
1. 前端调用 /register/send-code，传入手机号
2. 后端生成6位数字验证码
3. 后端调用 z-msg 发送短信
4. z-msg 从 z-meta 获取短信配置，发送验证码
5. 前端获取验证码后，调用 /register/phone
6. 后端验证验证码有效性
7. 验证通过后，创建用户记录，返回token
```

### 6.2 验证码生成规则
- 类型：数字
- 长度：6位
- 有效期：5分钟（可配置）
- 验证码存储时需加密或hash存储（可选，增强安全性）

### 6.3 验证码校验规则
- 验证码有效期内存放，可重复验证
- 验证码有效期过后，自动标记为过期
- 验证成功后，标记为已使用
- 同一接收者60秒内不能重复发送

## 七、配置说明

### 7.1 z-meta 配置项
在 z-meta 系统中配置以下字典/配置：
- `VERIFY_CODE_EXPIRE_MINUTES` - 验证码有效期（分钟）
- `VERIFY_CODE_LENGTH` - 验证码长度
- `VERIFY_CODE_SEND_INTERVAL` - 发送间隔（秒）
- `SMS_PROVIDER` - 短信服务商配置
- `EMAIL_SMTP_CONFIG` - 邮件SMTP配置

### 7.2 应用配置 (application.yml)
```yaml
z-ctc:
  verify-code:
    expire-minutes: 5
    length: 6
    send-interval: 60
```

## 八、安全考虑

1. **密码存储**：使用 BCrypt 或类似不可逆加密算法存储密码
2. **验证码防刷**：
   - 同一IP/手机号每日发送次数限制
   - 发送间隔限制
3. **登录防暴力破解**：
   - 连续失败5次后锁定账号15分钟
   - 图形验证码（可选）
4. **敏感信息**：手机号、邮箱等做脱敏处理返回
