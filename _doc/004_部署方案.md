# z-one-company 部署方案

## 1. 项目概述

z-one-company 是一个整合了多个微服务的综合管理平台，包含：

- **后端服务**: z-one-company-main-starter (整合了 z-config, z-ctc 等)
- **前端项目**: z-one-company-frontend (React + Umi + Ant Design)
- **Docker 部署**: docker/ 目录已存在基础配置

## 2. 架构设计

### 2.1 服务端口规划

| 服务 | 内部端口 | 映射端口 | Context Path | 说明 |
|------|---------|---------|--------------|------|
| z-config | 8888 | 8848 | /config | 配置中心服务 |
| z-ctc | 8888 | 8092 | /ctc | 认证中心服务 |
| z-wf | 8888 | 8091 | /wf | 工作流服务 |
| z-mist | 8888 | 8085 | /mist | 密钥管理服务 |
| z-meta | 8888 | 8093 | /meta | 元数据服务 |
| z-task | 8888 | 8090 | /task | 任务调度服务 |
| 前端页面 | 5173 (dev) | 80/443 (prod) | / | Nginx 静态服务 |

### 2.2 URL 路由规则

通过 Nginx 反向代理，实现统一入口：

```
www.example.com/           -> 前端静态页面
www.example.com/config/*   -> z-config 服务 (8848端口)
www.example.com/ctc/*      -> z-ctc 服务 (8092端口)
www.example.com/wf/*       -> z-wf 服务 (8091端口)
www.example.com/mist/*     -> z-mist 服务 (8085端口)
www.example.com/meta/*     -> z-meta 服务 (8093端口)
www.example.com/task/*     -> z-task 服务 (8090端口)
www.example.com/doc.html   -> API 文档 (任意服务)
```

### 2.2 统一域名访问

生产环境通过 Nginx 反向代理，实现统一域名访问：

```
www.example.com/           -> 前端页面 (80端口)
www.example.com/api/*      -> 后端服务 (8888端口)
www.example.com/doc.html   -> API 文档 (8888端口)
```

## 4. 构建配置

### 4.1 后端构建

在 `_build/` 目录下创建统一的 Dockerfile：

```dockerfile
FROM openjdk:8-jre-alpine

WORKDIR /app

# 复制 jar 包
COPY bootstraps/z-one-company-main-starter/target/z-one-company-main-starter-1.0.0-SNAPSHOT.jar app.jar

EXPOSE 8888

ENTRYPOINT ["java", "-jar", "app.jar"]
```

### 4.2 前端构建

前端使用 Vite 构建，输出静态文件到 `dist/` 目录。

Nginx 配置示例：

```nginx
server {
    listen 80;
    server_name localhost;

    # 前端静态文件
    location / {
        root /usr/share/nginx/html;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 代理
    location /api/ {
        proxy_pass http://backend:8888/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # API 文档
    location /doc.html {
        proxy_pass http://backend:8888/doc.html;
    }
}
```

## 8. 多服务部署模式 (当前+未来兼容)

### 核心原则：Controller 统一使用 `/api/模块名` 前缀

```java
@RequestMapping("/api/config")   // 配置中心
@RequestMapping("/api/ctc")      // 认证中心
@RequestMapping("/api/wf")       // 工作流
```

### 模式一：单应用模式 (当前)

所有服务打包到一个应用，通过 Nginx 简单代理：

```nginx
location /api/ {
    proxy_pass http://backend:8888/api/;
}
```

访问：`/api/config/xxx` → 配置中心

### 模式二：多容器模式 (未来)

每个服务独立部署，Nginx 按路径路由：

```nginx
location /config/ {
    proxy_pass http://z-config:8888/;
    rewrite ^/config/(.*) /$1 break;  # 去掉前缀
}
location /ctc/ {
    proxy_pass http://z-ctc:8888/;
    rewrite ^/ctc/(.*) /$1 break;
}
```

访问：`/config/xxx` → z-config 容器

### 兼容要点

1. Controller 路径固定为 `/api/模块名` → 单应用自动路由
2. 多容器时，Nginx 去掉前缀再转发 → 兼容单应用代码
3. 两套 Nginx 配置：`nginx.conf` (单应用) 和 `nginx.multi.conf` (多容器)
