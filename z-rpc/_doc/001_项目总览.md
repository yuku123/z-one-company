# 001_项目总览

## 1.1 模块定位

z-rpc（RPC 框架层）为所有模块间调用提供透明高性能的远程过程调用能力，使调用方像调用本地方法一样调用其他模块，屏蔽网络通信细节。

## 1.2 愿景

- **透明调用**：消费方引入接口包，像调用本地 Bean 一样调用远程服务
- **负载均衡**：RoundRobin、Random、加权、最小活跃数
- **失败重试**：超时重试、失败自动切换
- **服务发现**：从 z-gw 注册中心动态感知提供者地址
- **序列化**：JSON / Protobuf / Hession 高效序列化

## 1.3 模块关系

```
z-gw（注册中心）───服务注册──► z-rpc（消费方从注册中心发现）
z-ctc ──RPC──► z-task（跨模块调用）
z-task ──RPC──► z-wf（查询审批状态）
```

## 1.4 技术选型

| 方案 | 说明 |
|------|------|
| Dubbo | 高性能，成熟，国内生态好（推荐） |
| Spring Cloud Feign | 与 Spring Boot 生态无缝 |
| gRPC | 跨语言，高性能 |

## 1.5 开发优先级

| 优先级 | 内容 |
|--------|------|
| P0 | RPC 框架选型与基础骨架（推荐 Dubbo）、服务注册与消费 |
| P1 | 负载均衡策略、失败重试机制 |
| P2 | 序列化优化、流量控制 |
