# 001_项目总览

## 1.1 模块定位

z-ext（扩展模块）为整个平台提供插件化扩展能力，允许在不修改核心代码的情况下，通过注册扩展点动态加载外部业务逻辑，实现功能的即插即用。

## 1.2 愿景

- **扩展点机制**：定义清晰的扩展点接口，各模块实现扩展点注入自定义行为
- **插件生命周期**：插件的安装/启用/停用/卸载全周期管理
- **插件市场**：插件打包为 JAR，上传到平台，在线安装（规划中）
- **沙箱隔离**：插件运行在独立 ClassLoader，防止污染核心应用

## 1.3 模块关系

```
z-task ──扩展点──► z-ext（自定义任务字段类型、自定义提醒方式）
z-wf ──扩展点──► z-ext（自定义审批节点动作、自定义表单组件）
z-gw ──扩展点──► z-ext（自定义过滤器、自定义认证方式）
z-ctc ──鉴权──► z-ext（扩展点认证插件验证）
```

## 1.4 扩展点设计

```
ExtensionPoint（接口）
  └── TaskNotifier（任务通知扩展：邮件/钉钉/企微/飞书）
  └── ApprovalAction（审批动作扩展：加签/转交/委托）
  └── AuthPlugin（认证插件扩展：OAuth/SAML/CAS）
  └── CustomFilter（网关过滤器扩展）
```

## 1.5 技术栈

| 组件 | 版本 |
|------|------|
| Java | 8 |
| Spring Boot | 2.7.12 |
| SPI (java.util.ServiceLoader) | 插件发现 |
| 自定义 ClassLoader | 插件隔离 |

## 1.6 开发优先级

| 优先级 | 内容 |
|--------|------|
| P0 | 扩展点接口定义、Spring SPI 自动发现机制 |
| P1 | 插件 ClassLoader 隔离、插件元数据（manifest）规范 |
| P2 | 插件在线安装/卸载、插件市场 |
