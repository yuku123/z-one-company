# 扩展平台SDK - 产品方案设计

## 1. 产品定位

**扩展平台SDK** 是一个面向企业级应用的**模块化插拔架构解决方案**，旨在帮助技术团队实现：

- **平台与业务的解耦**：平台侧提供能力，业务侧提供实现
- **实现的热插拔**：无需重新部署即可切换接口实现
- **多场景适配**：一套代码兼容多种业务场景（自研/三方/定制）

---

## 2. 核心概念

### 2.1 角色定义

| 角色 | 说明 | 职责 |
|------|------|------|
| **平台方** | SDK 提供方 | 定义扩展点、默认实现、路由规则 |
| **业务方** | SDK 使用方 | 实现/替换扩展点、配置路由 |
| **运维方** | 部署/运维 | 动态切换、监控配置 |

### 2.2 核心术语

- **扩展点 (Extension Point)**：被标记的接口，定义能力边界
- **实现 (Implementation)**：扩展点的具体业务逻辑
- **路由 (Route)**：决定使用哪个实现的规则
- **代理 (Proxy)**：拦截调用并分发的运行时对象

## 4. 使用场景

### 4.1 场景一：三方服务替换

**场景描述**：原系统使用本地支付能力，现在需要接入支付宝/微信支付。

```
平台侧定义：
┌────────────────────────────────────┐
│  @ExtensionPoint("payment")       │
│  interface PaymentService {       │
│      PayResult pay(PayRequest);   │
│  }                                │
│                                    │
│  @Extension impl=PLATFORM         │
│  class DefaultPayment {...}       │
└────────────────────────────────────┘

业务方替换：
┌────────────────────────────────────┐
│  @Extension impl=EXTERNAL         │
│  class AlipayPayment implements   │
│      PaymentService {...}         │
└────────────────────────────────────┘

效果：无需修改业务调用代码，SDK 自动路由到支付宝实现
```

### 4.2 场景二：多环境适配

**场景描述**：开发环境用本地实现，测试/生产环境用三方实现。

```yaml
# 配置中心或本地配置
extension:
  route:
    - point: "user.sync"
      condition: "env == 'prod'"
      target: external
    - point: "user.sync"
      condition: "env != 'prod'"
      target: platform
```

### 4.3 场景三：灰度发布

**场景描述**：新版本实现需要灰度验证。

```yaml
extension:
  route:
    - point: "order.service"
      condition: "user.grade == 'VIP'"
      target: external-v2
      weight: 10%
```

### 4.4 场景四：故障切换

**场景描述**：三方服务故障时，自动切换到本地实现。

```java
// 策略配置
@Extension(
    point = "order.create",
    fallback = "platform",  // 三方失败时回退
    timeout = 3000,         // 超时阈值
    retry = 3               // 重试次数
)
class ExternalOrderService implements OrderCreateService {
    // ...
}
```

## 8. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 代理性能开销 | 延迟增加 | 使用字节码增强，优化路由缓存 |
| 路由配置复杂度 | 使用门槛高 | 提供IDE插件和配置生成器 |
| 故障传播 | 单点故障影响全局 | 实现熔断和降级机制 |
| 升级兼容性 | SDK升级破坏业务 | 严格版本管理，向后兼容
