## 一、MCP 协议概述

### 1.1 什么是 MCP

MCP（Model Context Protocol，模型上下文协议）是由 Anthropic 提出的开放协议，旨在标准化 AI 模型与外部数据源、工具之间的集成方式。MCP 定义了一套标准化的接口和消息格式，使得 AI 应用能够方便地发现和调用各种工具。

### 1.2 MCP 核心概念

| 概念 | 说明 |
|------|------|
| **Server** | 提供工具、资源、提示词的服务端 |
| **Client** | 连接并使用 Server 的客户端（如 Claude Desktop）|
| **Tool** | 可执行的功能（如搜索、计算、API 调用）|
| **Resource** | 可被读取的数据源（如文件、数据库）|
| **Prompt** | 预定义的提示词模板 |
| **Sampling** | 服务端请求客户端进行 LLM 采样 |

### 1.3 MCP 协议版本

当前实现基于 **MCP 2024-11-05** 协议版本。

协议版本说明：
- 协议版本格式为 `YYYY-MM-DD`
- 客户端和服务器在初始化时协商协议版本
- 如果版本不匹配，使用双方支持的最高共同版本

### 1.4 MCP 通信模型

MCP 支持两种通信方式：

#### 1.4.1 Stdio 传输（标准输入输出）
- 基于进程间通信
- 适用于本地工具集成
- 消息格式为 JSON-RPC 2.0

#### 1.4.2 SSE/HTTP 传输（服务器发送事件）
- 基于 HTTP 协议
- 适用于远程服务
- 支持流式响应
- 支持双向通信

## 二、MCP 协议详解

### 2.1 JSON-RPC 2.0 消息格式

MCP 使用 JSON-RPC 2.0 作为消息格式。

#### 2.1.1 请求格式

```json
{
  "jsonrpc": "2.0",
  "id": "request-id-123",
  "method": "tools/list",
  "params": {
    "filter": "optional-filter"
  },
  "_meta": {
    "progressToken": "progress-123"
  }
}
```

字段说明：
- `jsonrpc`: 固定为 "2.0"
- `id`: 请求唯一标识（通知可为 null）
- `method`: 调用的方法名
- `params`: 方法参数（可选）
- `_meta`: 元数据（如进度通知 token）

#### 2.1.2 响应格式

成功响应：
```json
{
  "jsonrpc": "2.0",
  "id": "request-id-123",
  "result": {
    "tools": [...]
  }
}
```

错误响应：
```json
{
  "jsonrpc": "2.0",
  "id": "request-id-123",
  "error": {
    "code": -32601,
    "message": "Method not found",
    "data": null
  }
}
```

#### 2.1.3 通知格式

通知是没有 id 的请求：
```json
{
  "jsonrpc": "2.0",
  "method": "notifications/progress",
  "params": {
    "token": "progress-123",
    "progress": 50,
    "total": 100
  }
}
```

### 2.2 MCP 标准方法

#### 2.2.1 生命周期方法

| 方法 | 说明 |
|------|------|
| `initialize` | 初始化连接，协商协议版本和能力 |
| `ping` | 心跳检测 |
| `notifications/initialized` | 初始化完成通知 |

#### 2.2.2 工具方法

| 方法 | 说明 |
|------|------|
| `tools/list` | 获取可用工具列表 |
| `tools/call` | 调用指定工具 |
| `notifications/tools/list_changed` | 工具列表变更通知 |

#### 2.2.3 资源方法

| 方法 | 说明 |
|------|------|
| `resources/list` | 获取可用资源列表 |
| `resources/read` | 读取指定资源 |
| `resources/subscribe` | 订阅资源变更 |
| `resources/unsubscribe` | 取消资源订阅 |
| `notifications/resources/updated` | 资源更新通知 |
| `notifications/resources/list_changed` | 资源列表变更通知 |

#### 2.2.4 提示词方法

| 方法 | 说明 |
|------|------|
| `prompts/list` | 获取可用提示词列表 |
| `prompts/get` | 获取指定提示词 |
| `notifications/prompts/list_changed` | 提示词列表变更通知 |

#### 2.2.5 采样方法（服务端→客户端）

| 方法 | 说明 |
|------|------|
| `sampling/createMessage` | 请求客户端创建消息（LLM 采样）|

#### 2.2.6 根路径方法

| 方法 | 说明 |
|------|------|
| `roots/list` | 获取根路径列表 |

### 2.3 MCP 数据模型

#### 2.3.1 Tool 定义

```json
{
  "name": "search",
  "description": "Search for information",
  "inputSchema": {
    "type": "object",
    "properties": {
      "query": {
        "type": "string",
        "description": "Search query"
      }
    },
    "required": ["query"]
  },
  "annotations": {
    "readOnlyHint": false,
    "idempotentHint": false,
    "destructiveHint": false
  }
}
```

#### 2.3.2 Resource 定义

```json
{
  "uri": "file:///path/to/file.txt",
  "name": "Example File",
  "description": "An example text file",
  "mimeType": "text/plain",
  "size": 1024
}
```

#### 2.3.3 Prompt 定义

```json
{
  "name": "code-review",
  "description": "Review code for issues",
  "arguments": {
    "properties": [
      {
        "name": "language",
        "type": "string",
        "description": "Programming language"
      }
    ],
    "required": ["language"]
  }
}
```

#### 2.3.4 Content 类型

MCP 支持多种内容类型：

```json
// 文本内容
{
  "type": "text",
  "text": "Hello, world!"
}

// 图片内容
{
  "type": "image",
  "mimeType": "image/png",
  "data": "base64-encoded-data"
}

// 资源引用
{
  "type": "resource",
  "uri": "file:///path/to/file",
  "mimeType": "text/plain"
}
```

### 2.4 MCP 错误码

MCP 使用 JSON-RPC 2.0 标准错误码：

| 错误码 | 名称 | 说明 |
|--------|------|------|
| -32700 | Parse error | 解析错误 |
| -32600 | Invalid Request | 无效请求 |
| -32601 | Method not found | 方法未找到 |
| -32602 | Invalid params | 无效参数 |
| -32603 | Internal error | 内部错误 |
| -32000 ~ -32099 | Server error | 服务端错误（自定义）|

### 3.4 协议实现详情

#### 3.4.1 协议常量定义

`McpProtocolConstants.java` 定义了所有 MCP 协议相关的常量：

```java
// 协议版本
public static final String PROTOCOL_VERSION = "2024-11-05";
public static final String JSON_RPC_VERSION = "2.0";

// 标准方法名
tools/list, tools/call
resources/list, resources/read
prompts/list, prompts/get
initialize, ping
```

#### 3.4.2 请求/响应格式

**McpRequestV1** 结构：
```java
@Data
public class McpRequestV1 {
    private String jsonrpc = "2.0";
    private Object id;              // 请求ID
    private String method;          // 方法名
    private Map<String, Object> params;  // 参数
    private RequestMeta _meta;      // 元数据（如 progressToken）
}
```

**McpResponseV1** 结构：
```java
@Data
public class McpResponseV1 {
    private String jsonrpc = "2.0";
    private Object id;              // 对应请求ID
    private Object result;          // 成功结果
    private Error error;            // 错误信息
    private ResponseMeta _meta;     // 元数据
}
```

#### 3.4.3 会话管理

**McpSession** 结构：
```java
@Data
public class McpSession {
    private String sessionId;           // 会话ID
    private String status;              // 状态（initializing/active/closing/closed）
    private ClientCapabilities clientCapabilities;  // 客户端能力
    private String protocolVersion;     // 协商的协议版本
    private long createTime;            // 创建时间
    private long lastActivityTime;      // 最后活动时间
    private Set<String> subscribedResources;  // 订阅的资源
    private Map<String, ProgressTracker> progressTrackers;  // 进度追踪
}
```

会话状态流转：
```
initializing → active → closing → closed
```

### 3.6 核心组件实现

#### 3.6.1 McpProtocolService

`McpProtocolService` 是 MCP 协议的核心处理器，负责处理所有标准 MCP 请求。

**主要职责：**
- 初始化处理 (`initialize`)
- 心跳检测 (`ping`)
- 工具管理 (`tools/list`, `tools/call`)
- 资源管理 (`resources/list`, `resources/read`)
- 提示词管理 (`prompts/list`, `prompts/get`)

**处理流程：**
```java
public McpResponseV1 handleRequest(McpRequestV1 request, McpSession session) {
    switch (method) {
        case METHOD_INITIALIZE:
            return handleInitialize(request, session);
        case METHOD_TOOLS_LIST:
            return handleToolsList(request, session);
        case METHOD_TOOLS_CALL:
            return handleToolsCall(request, session);
        // ... 其他方法
    }
}
```

#### 3.6.2 McpRegistry

`McpRegistry` 是工具注册中心，管理所有可用的工具。

**主要职责：**
- 工具注册和注销
- 工具元数据管理
- 内置工具与第三方工具区分
- 工具发现

**工具分类：**
- **内置工具** (`BUILT_IN`): 由系统提供，不可外部调用
- **第三方工具** (`THIRD_PARTY`): 由外部服务提供，可动态调用

#### 3.6.3 McpHandler

`McpHandler` 是工具执行处理器，负责实际调用工具。

**主要职责：**
- 内置工具执行
- 第三方工具 HTTP 调用
- 结果转换

#### 3.6.4 SseMcpController

`SseMcpController` 实现了 SSE（Server-Sent Events）流式通信。

**主要职责：**
- SSE 连接管理
- 事件推送
- 心跳维护

**事件类型：**
- `connected`: 连接建立
- `message`: JSON-RPC 消息
- `progress`: 进度通知
- `error`: 错误通知
- `heartbeat`: 心跳

### 3.8 配置与扩展
