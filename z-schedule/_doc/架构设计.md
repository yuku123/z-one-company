# Z-Schedule 架构设计文档

## 1. 项目概述

Z-Schedule 是一个分布式任务调度平台，对标 XXL-JOB，专注于解决分布式系统中的定时任务调度问题。其核心设计目标是开发迅速、学习简单、轻量级、易扩展。

### 1.1 核心特性

- **简单灵活**: 支持通过 Web 页面对任务进行 CRUD 操作，支持动态修改任务状态、暂停/恢复任务
- **动态调度**: 支持多种任务调度策略（Cron、固定间隔、一次性任务）
- **执行器自动注册**: 执行器自动注册到调度中心，支持动态扩容缩容
- **负载均衡**: 任务执行支持多种负载均衡策略（轮询、随机、一致性哈希等）
- **故障转移**: 任务执行失败支持重试、故障转移
- **分片广播**: 支持分片任务，一个大任务分片到多个执行器并行执行
- **阻塞处理策略**: 支持串行、丢弃、覆盖等多种阻塞处理策略
- **监控告警**: 任务执行记录、日志、失败告警

#### 3.1.3 调度引擎设计

```java
// 调度引擎核心组件
public class JobTrigger {

    /**
     * 触发任务执行
     */
    public void trigger(int jobId, TriggerTypeEnum triggerType, int failRetryCount, String executorShardingParam) {
        // 1. 加载任务信息
        JobInfo jobInfo = jobInfoDao.loadById(jobId);

        // 2. 检查任务状态
        if (jobInfo.getStatus() != JobStatusEnum.RUNNING) {
            return;
        }

        // 3. 生成任务日志
        JobLog jobLog = new JobLog();
        jobLog.setJobId(jobId);
        jobLog.setTriggerTime(new Date());
        jobLog.setTriggerCode(TriggerCodeEnum.SUCCESS);
        jobLogDao.save(jobLog);

        // 4. 获取执行器地址列表
        List<String> addressList = getExecutorAddresses(jobInfo.getAppName());

        // 5. 路由策略选择执行器
        String address = XxlJobTrigger.router(jobInfo.getExecutorRouteStrategy(), addressList);

        // 6. 触发远程执行
        runExecutor(triggerParam, address);
    }
}
```

#### 3.2.3 任务处理器设计

```java
/**
 * 任务处理器接口
 * 业务方实现此接口来定义具体的任务逻辑
 */
public interface IJobHandler {

    /**
     * 执行任务
     * @param triggerParam 触发参数
     * @return 执行结果
     * @throws Exception 执行异常
     */
    ReturnT<String> execute(TriggerParam triggerParam) throws Exception;

    /**
     * 初始化回调（可选）
     */
    default void init() throws Exception {}

    /**
     * 销毁回调（可选）
     */
    default void destroy() throws Exception {}
}

/**
 * 业务示例：定时数据同步任务
 */
@JobHandler(value = "dataSyncJob")
@Component
public class DataSyncJobHandler implements IJobHandler {

    @Autowired
    private DataService dataService;

    @Override
    public ReturnT<String> execute(TriggerParam param) throws Exception {
        // 1. 解析参数
        String tableName = param.getExecutorParams();

        // 2. 执行业务逻辑
        int count = dataService.syncData(tableName);

        // 3. 返回结果
        return ReturnT.success("同步完成，共处理 " + count + " 条记录");
    }
}
```

## 11. 总结

### 11.1 核心设计思想

1. **简单至上**: 架构设计力求简单，降低学习和维护成本
2. **解耦分离**: 调度中心与执行器分离，支持独立部署和扩展
3. **高可用**: 无单点设计，支持故障自动转移
4. **可观测**: 完善的监控指标和日志追踪

### 11.2 技术选型参考

| 层次 | 技术选型 | 说明 |
|------|---------|------|
| 调度中心 | Spring Boot + Vue | 快速开发，生态丰富 |
| 数据库 | MySQL 5.7+ | 成熟稳定，支持主从 |
| 通信协议 | HTTP / gRPC | 简单通用 / 高性能 |
| 注册中心 | 数据库 | 简单场景，避免引入额外组件 |
| 监控 | Prometheus + Grafana | 云原生监控标准 |
| 告警 | AlertManager / 钉钉/企微 | 多渠道告警 |

### 11.3 演进路线

```
Phase 1 (MVP): 基础调度功能
  - 任务CRUD、Cron调度、执行器注册、任务执行、日志记录

Phase 2 (增强): 高可用与性能
  - 集群部署、路由策略、故障转移、性能优化

Phase 3 (企业): 企业级功能
  - 权限管理、审计日志、GLUE脚本、工作流编排

Phase 4 (云原生): 云原生支持
  - Kubernetes Operator、Serverless执行器、多云调度
```
