# 001_项目总览

## 1.1 模块定位

z-schedule 是**分布式任务调度平台**，为系统提供统一的定时任务调度、执行和监控能力。支持分布式执行器集群、负载均衡、故障转移、分片广播等特性。

## 1.2 愿景

为一人公司提供生产级的分布式任务调度能力：
- **简单灵活**：通过 Web 页面即可完成任务的 CRUD 操作，支持动态修改任务状态
- **分布式执行**：执行器自动注册到调度中心，支持动态扩缩容
- **高可用**：任务失败自动重试，支持故障转移和阻塞处理策略
- **分片并行**：支持分片任务，一个大任务分片到多个执行器并行执行

## 1.3 模块关系

```
业务服务（依赖 z-schedule-executor SDK）───注册&执行──► z-schedule-admin（调度中心）
                                                        │
                                                    调度触发
                                                        │
z-ctc（用户/权限/审计）◄───管理操作── z-schedule-admin ◄─┘
```

## 1.4 服务端口

| 部署方式 | 端口 |
|---------|------|
| z-schedule-admin | 8080 |
| 执行器（嵌入业务服务） | 随机 |

## 1.5 模块结构

```
z-schedule/
├── z-schedule-core/          # 核心模块（公共实体、API、枚举等）
├── z-schedule-admin/        # 调度中心（Web管理 + 调度引擎）
└── z-schedule-executor/     # 执行器 SDK（供业务系统依赖）
```

## 1.6 核心特性

| 特性 | 说明 |
|------|------|
| 动态调度 | 支持 Cron、固定间隔、一次性任务 |
| 执行器注册 | 执行器自动注册到调度中心 |
| 负载均衡 | 轮询、随机、一致性哈希等策略 |
| 故障转移 | 失败重试 + 故障转移 |
| 分片广播 | 分片任务并行执行 |
| 阻塞处理 | 串行、丢弃、覆盖等策略 |
| 监控告警 | 执行日志、失败告警 |
