# z-meta 元数据平台设计文档

## 一、系统概述

### 1.1 背景与定位
z-meta 是面向 SaaS 多租户场景的元数据管理平台，为 z-suit 部署套件提供应用、接口、字典等元数据的统一管理能力。

### 1.2 核心能力
- 多租户支持：租户域隔离
- 应用管理：租户域下的应用注册与配置
- 字典管理：租户域下的数据字典与枚举值
- 接口清单：应用下的 API 接口清单管理

### 1.3 技术选型参考
- 后端：Java + Spring Boot + MyBatis-Plus（参考 z-config）
- 前端：React 19 + Ant Design 6（参考 z-config-admin-frontend）
- 架构模式：分层架构（Controller/Service/Mapper）

## 四、API 接口设计

### 4.1 租户域管理
- `POST /api/tenant` - 创建租户
- `GET /api/tenant/{id}` - 获取租户详情
- `PUT /api/tenant/{id}` - 更新租户
- `DELETE /api/tenant/{id}` - 删除租户
- `GET /api/tenant/list` - 租户列表

### 4.2 应用管理
- `POST /api/app` - 创建应用
- `GET /api/app/{id}` - 获取应用详情
- `PUT /api/app/{id}` - 更新应用
- `DELETE /api/app/{id}` - 删除应用
- `GET /api/app/list` - 应用列表（支持租户筛选）

### 4.3 字典管理
- `POST /api/dict/type` - 创建字典类型
- `GET /api/dict/type/{id}` - 获取字典类型
- `PUT /api/dict/type/{id}` - 更新字典类型
- `GET /api/dict/type/list` - 字典类型列表
- `POST /api/dict/item` - 创建字典项
- `GET /api/dict/item/{dictId}` - 获取字典项列表
- `PUT /api/dict/item/{id}` - 更新字典项
- `DELETE /api/dict/item/{id}` - 删除字典项
- `GET /api/dict/items/{dictCode}` - 根据编码获取字典项

### 4.4 接口清单
- `POST /api/interface` - 创建接口
- `GET /api/interface/{id}` - 获取接口详情
- `PUT /api/interface/{id}` - 更新接口
- `DELETE /api/interface/{id}` - 删除接口
- `GET /api/interface/list` - 接口列表（支持应用筛选）
- `GET /api/interface/search` - 接口搜索

## 八、与 z-suit 集成

z-meta 将作为 z-suit 配置的一部分进行部署：
1. 在 `repos.yaml` 中添加 z-meta 项目
2. z-suit 自动拉取代码、编译、构建镜像
3. 启动 z-meta 容器并配置路由
4. 通过 z-suit dashboard 访问 z-meta 管理界面
