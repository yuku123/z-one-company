# z-meta 元数据平台技术方案

## 一、技术架构

### 1.1 技术栈
| 层级 | 技术选型 |
|------|----------|
| 后端框架 | Spring Boot 3.x |
| 数据访问 | MyBatis-Plus 3.5.x |
| 数据库 | MySQL 8.x |
| 前端框架 | React 19.x + react-router-dom |
| UI组件 | Ant Design 6.x |
| 构建工具 | Vite 8.x |
| 服务注册 | 内部服务（参考 z-config） |

### 1.2 架构图

```

│                         浏览器/客户端                            │
```

## 三、数据库设计

### 3.1 核心表结构

```sql
-- 租户域表
CREATE TABLE `z_tenant` (
    `id` bigint(20) NOT NULL AUTO_INCREMENT,
    `tenant_code` varchar(64) NOT NULL COMMENT '租户编码',
    `tenant_name` varchar(128) NOT NULL COMMENT '租户名称',
    `tenant_type` varchar(32) DEFAULT 'enterprise' COMMENT '租户类型',
    `status` tinyint(1) DEFAULT 1 COMMENT '状态',
    `expire_time` datetime DEFAULT NULL COMMENT '过期时间',
    `logo_url` varchar(500) DEFAULT NULL COMMENT 'Logo',
    `description` varchar(500) DEFAULT NULL COMMENT '描述',
    `gmt_create` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP,
    `gmt_modified` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_tenant_code` (`tenant_code`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='租户域表';

-- 应用表
CREATE TABLE `z_application` (
    `id` bigint(20) NOT NULL AUTO_INCREMENT,
    `tenant_id` bigint(20) NOT NULL COMMENT '租户ID',
    `app_code` varchar(64) NOT NULL COMMENT '应用编码',
    `app_name` varchar(128) NOT NULL COMMENT '应用名称',
    `app_type` varchar(32) DEFAULT 'web' COMMENT '应用类型',
    `description` varchar(500) DEFAULT NULL COMMENT '描述',
    `base_url` varchar(255) DEFAULT NULL COMMENT '基础URL',
    `status` tinyint(1) DEFAULT 1 COMMENT '状态',
    `owner` varchar(64) DEFAULT NULL COMMENT '负责人',
    `gmt_create` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP,
    `gmt_modified` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_app_code_tenant` (`app_code`,`tenant_id`),
    KEY `idx_tenant_id` (`tenant_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='应用表';

-- 字典类型表
CREATE TABLE `z_dict_type` (
    `id` bigint(20) NOT NULL AUTO_INCREMENT,
    `tenant_id` bigint(20) NOT NULL COMMENT '租户ID',
    `dict_code` varchar(64) NOT NULL COMMENT '字典编码',
    `dict_name` varchar(128) NOT NULL COMMENT '字典名称',
    `dict_type` varchar(32) DEFAULT 'custom' COMMENT '字典类型',
    `status` tinyint(1) DEFAULT 1 COMMENT '状态',
    `description` varchar(500) DEFAULT NULL COMMENT '描述',
    `gmt_create` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP,
    `gmt_modified` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_dict_code_tenant` (`dict_code`,`tenant_id`),
    KEY `idx_tenant_id` (`tenant_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='字典类型表';

-- 字典项表
CREATE TABLE `z_dict_item` (
    `id` bigint(20) NOT NULL AUTO_INCREMENT,
    `dict_id` bigint(20) NOT NULL COMMENT '字典类型ID',
    `item_code` varchar(64) NOT NULL COMMENT '字典项编码',
    `item_name` varchar(128) NOT NULL COMMENT '字典项名称',
    `item_value` varchar(255) NOT NULL COMMENT '字典项值',
    `sort_order` int(11) DEFAULT 0 COMMENT '排序',
    `status` tinyint(1) DEFAULT 1 COMMENT '状态',
    `parent_id` bigint(20) DEFAULT NULL COMMENT '父级ID',
    `ext_props` text COMMENT '扩展属性',
    `gmt_create` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP,
    `gmt_modified` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (`id`),
    KEY `idx_dict_id` (`dict_id`),
    KEY `idx_parent_id` (`parent_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='字典项表';

-- 接口清单表
CREATE TABLE `z_api` (
    `id` bigint(20) NOT NULL AUTO_INCREMENT,
    `app_id` bigint(20) NOT NULL COMMENT '应用ID',
    `api_path` varchar(255) NOT NULL COMMENT '接口路径',
    `api_method` varchar(16) DEFAULT 'GET' COMMENT '请求方法',
    `api_name` varchar(128) NOT NULL COMMENT '接口名称',
    `api_version` varchar(32) DEFAULT 'v1' COMMENT '接口版本',
    `description` varchar(500) DEFAULT NULL COMMENT '描述',
    `request_params` text COMMENT '请求参数JSON',
    `response_params` text COMMENT '响应参数JSON',
    `auth_type` varchar(32) DEFAULT 'bearer' COMMENT '认证类型',
    `status` tinyint(1) DEFAULT 1 COMMENT '状态',
    `deprecated` tinyint(1) DEFAULT 0 COMMENT '是否废弃',
    `gmt_create` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP,
    `gmt_modified` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (`id`),
    KEY `idx_app_id` (`app_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='接口清单表';
```

## 六、安全设计

### 6.1 租户隔离
- 所有数据操作强制带 tenant_id 条件
- Service 层统一进行租户校验
- Mapper 层进行租户 ID 注入

### 6.2 认证机制
- JWT Token 认证
- Token 包含：userId, tenantId, roles, exp
- 请求头：Authorization: Bearer <token>

## 七、集成部署

### 7.1 与 z-suit 集成

在 z-suit 的 `configs/repos.yaml` 中添加：
```yaml
repos:
  - name: z-meta
    git_url: git@github.com:your-org/z-meta.git
    branch: main
    type: java
    port: 8080
    context_path: /meta
    enabled: true
```

### 7.2 路由配置
- 访问路径：`http://domain.com/meta`
- API 路径：`http://domain.com/meta/api/...`
